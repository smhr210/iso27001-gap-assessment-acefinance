# Assessment Methodology

## Overview
This assessment uses a two-dimensional scoring model combining control maturity rating with a Likelihood x Impact risk score to prioritise remediation.

## Maturity Scale (0–4)

| Score | Level | Description |
|---|---|---|
| 0 | Not Implemented | Control does not exist |
| 1 | Ad Hoc | Control exists informally but is undocumented and inconsistently applied |
| 2 | Defined | Control is defined and documented but not consistently enforced |
| 3 | Implemented | Control is implemented, documented, and consistently applied |
| 4 | Optimised | Control is monitored, measured, and continuously improved |

## Risk Scoring Model

Risk Score = Likelihood (1–5) × Impact (1–5)

### Likelihood Scale
1 = Rare, 2 = Unlikely, 3 = Possible, 4 = Likely, 5 = Almost Certain

### Impact Scale
1 = Negligible, 2 = Minor, 3 = Moderate, 4 = Major, 5 = Severe

### Risk Rating Thresholds
| Score | Rating | Remediation Timeline |
|---|---|---|
| 1–6 | Low | Monitor — remediate opportunistically |
| 7–12 | Medium | Remediate within 90 days |
| 13–16 | High | Remediate within 60 days |
| 17–25 | Critical | Remediate within 30 days |

## Evidence Sources
- Existing network architecture documentation (Project 4 — Network Security Docs)
- Vendor contracts and policy documentation review
- Process interviews with fictional department leads
- Comparison against ISO/IEC 27001:2022 Annex A requirements

## Limitations
This is a simulated assessment conducted for portfolio purposes using a fictional organization. All findings, risks, and recommendations are illustrative.
